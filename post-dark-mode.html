Building a Dark Mode Toggle with Vanilla JavaScript and localStorage

📅 April 22, 2025 ✍️ Muhammad Syamil ☕ 3 min read

The dark mode toggle on this blog does three things: it respects your system preference on first visit, saves your choice to localStorage, and applies the theme instantly on every subsequent page load to prevent a "flash of wrong theme." Here's exactly how it's built.

Step 1 — CSS Custom Properties

Define all your colours as CSS variables (custom properties) on the :root selector, then override them inside a [data-theme="dark"] attribute selector.

/* Light theme (default) */ :root { --bg: #faf9f6; --text: #1c1917; --accent: #c2410c; } /* Dark theme — override via attribute */ [data-theme="dark"] { --bg: #0f0e0c; --text: #f5f0eb; --accent: #fb923c; }

Every element in your CSS should use these variables instead of raw colours: background: var(--bg), color: var(--text). When you swap the attribute, everything updates instantly.

Step 2 — The Toggle Button

<button id="themeToggle" aria-label="Toggle dark mode"> <span id="themeIcon">🌙</span> <span id="themeLabel">Dark</span> </button>

Step 3 — JavaScript Logic

const themeToggle = document.getElementById('themeToggle'); const themeIcon = document.getElementById('themeIcon'); const themeLabel = document.getElementById('themeLabel'); function applyTheme(theme) { // Set the attribute on <html> document.documentElement.setAttribute('data-theme', theme); // Persist to localStorage localStorage.setItem('blog-theme', theme); // Update the button UI themeIcon.textContent = theme === 'dark' ? '☀️' : '🌙'; themeLabel.textContent = theme === 'dark' ? 'Light' : 'Dark'; } // On first visit: check localStorage, then system preference const saved = localStorage.getItem('blog-theme'); const system = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light'; applyTheme(saved || system); // Toggle on button click themeToggle.addEventListener('click', () => { const current = document.documentElement.getAttribute('data-theme'); applyTheme(current === 'dark' ? 'light' : 'dark'); });

Why This Approach Works Well

The code runs immediately on page load (not inside DOMContentLoaded), so the theme is applied before the browser paints. This prevents the flash of white that appears on dark-mode sites when the theme is applied too late.

Using data-theme on the <html> element means the CSS override applies to every descendant — you don't need to pass the theme through props or context like you would in a framework.

Finally, window.matchMedia('(prefers-color-scheme: dark)') reads the user's operating system preference. This means first-time visitors immediately get the theme they expect, without any interaction.

💡 Try it now: Click the 🌙 button in the top right. Then refresh the page — your preference is saved. That's localStorage at work.
