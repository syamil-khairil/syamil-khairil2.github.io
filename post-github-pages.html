Getting Started with GitHub Pages: Deploy for Free in 10 Minutes

📅 May 1, 2025 ✍️ Muhammad Syamil ☕ 4 min read

When I finished my first HTML project, I had no idea how to show it to anyone. It lived on my laptop, inside a folder, going nowhere. Then a classmate mentioned GitHub Pages — and within ten minutes, my project was live on the internet with a real URL. For free. No server, no hosting fees, no configuration files.

Here's exactly how to do it.

What Is GitHub Pages?

GitHub Pages is a free static site hosting service built into GitHub. Any repository containing HTML, CSS, or JavaScript files can be published as a website. Your URL will look like: yourusername.github.io/repo-name.

Step-by-Step Guide

Create a GitHub account

Go to github.com and sign up if you haven't already. Choose a username that looks professional — it'll appear in your URL.

Create a new repository

Click the green New button on your GitHub dashboard. Name the repository (e.g., personal-blog), set it to Public, and click Create. Don't add a README yet.

Upload your project files

In the empty repository, click uploading an existing file and drag your project folder in. Or use Git in the terminal:

git init
git add .
git commit -m "Initial commit: Add project files"
git branch -M main
git remote add origin https://github.com/syamil-khairil/personal-blog.git
git push -u origin main
Enable GitHub Pages

Go to your repository → Settings → Pages (in the left sidebar). Under Source, select Deploy from a branch, then choose main / root. Click Save.

Wait 60 seconds and visit your site

GitHub will build and deploy your site. After about a minute, your site is live at:

https://syamil-khairil.github.io/personal-blog/

If it doesn't load immediately, wait a couple of minutes and refresh. The first deployment takes slightly longer.

Tips for a Smooth Deployment

Make sure your entry point is named index.html (not home.html or anything else). GitHub Pages serves index.html at the root URL.

Use relative paths for your CSS and JS files (./css/style.css instead of absolute paths), or they'll break when deployed to a subfolder URL.

Every time you push a new commit to the main branch, GitHub Pages automatically rebuilds and redeploys. No manual steps needed after setup.

What You Just Did

You deployed a real website, accessible to anyone in the world, for zero cost. GitHub Pages is how this blog is hosted. It's also how countless open source project documentation sites, portfolios, and landing pages are served every day.

🚀 Next step: Add a custom domain. GitHub Pages supports free custom domains — you just need to buy the domain (~RM40/year) and configure a CNAME record. GitHub's documentation covers this in detail.
